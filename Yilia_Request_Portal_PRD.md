# PRD --- Yilia Request Portal

## 1. Overview

### Background

Design requests are often sent through chat, making them difficult to
track and prioritize. This project creates a lightweight, playful
request portal that encourages teammates to submit requests in a
structured way while keeping the experience memorable.

### Goal
- Collect design requests through a simple web page.
- Automatically write submissions into a Feishu Bitable.
- Add a small amount of personality without reducing usability.
- Keep the submission flow under one minute.

---

# 2. User Flow

```
Landing Page
    ├── 提需求
    │      ↓
    │   Request Form
    │      ↓
    │  Submit (immediately)
    │      ↓
    │ Write to Feishu Bitable
    │      ↓
    └── Success Receipt
           ↓
      Toast
```

---

# 3. Landing Page

## Title

**哈喽，你有什么事呀～**

Two buttons:
- 提需求
- Just say hi

### 提需求

Interaction:
- Button lightly avoids the cursor.
- Movement is limited to a small nearby area.
- Escapes only 2--3 times.
- Afterwards it becomes easy to click.
- Mobile devices always display a normal clickable button.

Goal: Create a fun first impression without frustrating users.

### 说点什么

Click behavior:

Show a toast only.

Example:

> 收到啦 👋 祝你今天开心！
- No page transition.
- No data stored.
- Toast disappears automatically after ~2--3 seconds.

# 4. Request Form

## Required Field

### 需求内容

Type: Multi-line textarea

Placeholder:

> 需要我做什么～

---

## Optional Fields

### 你的名字

Text input

---

### 截止时间

Text input or Date picker (allow user to write freely, or also can click calendar icon to choose)

---

### 优先级

Pills Options:
- 不着急
- 正常
- 有点急
- 十万火急！

No default selection.

---

## Submit Button

Text:

**提交**

Behavior:
- Immediately submits.
- No confirmation dialog.
- Disable button while submitting.
- Show loading state:
  - 正在提交...

---

# 5. Success Receipt

After successful submission:

## Title

**你的需求已提交 ✓**

Subtitle:

> 小亚已经收到，会尽快查看。

Display the submitted information exactly as entered.

Example:

### 需求内容

input example: 优化设计

### 你的名字

Alex

### Deadline

2026-07-20

### 优先级

正常

Rules:
- Hide optional fields that were left empty.
- Do not display "未填写".
- Data has already been submitted before this screen appears.

Buttons:
- 再提一个需求
- 返回首页

# 7. Backend

## Architecture

```
HTML
    ↓
Cloudflare Worker
    ↓
Feishu Open API
    ↓
Feishu Bitable
```

Responsibilities:

Cloudflare Worker
- Validate request
- Call Feishu API
- Return success/failure

Feishu Bitable
- Store requests
- Track status

---

# 8. UX Principles
- Playful but respectful.
- One joke is enough.
- Never block task submission.
- Minimize required inputs.
- Clear submission feedback.
- Make users confident their request has been received.
